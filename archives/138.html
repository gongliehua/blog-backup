腾讯云主机 & VPS & 信用卡

这个博客用的服务器使用的是腾讯云的香港主机（1c1h）

有时候用 xshell 连着明显的感觉到卡。我想着腾讯云后台能监控CPU什么的应该在这个服务器上安装了监控软件的，1c1h的小主机还给安装防护及监控软件，浪费资源，要是真被攻击了也抗不了啥。经过网上搜索，找到了卸载办法，卸载后感觉好一些了，腾讯云后台的监控也断了。

我之前的环境跑在 docker 中的，内存太少了，我觉得没必要跑 docker，再加上也想把系统重置清理一下，所以就重新装了系统，其实腾讯云后台重新安装系统时，下方是会默认把 安装doos防护和监控软件 打勾的，去掉就好了。
重装好后，连上系统 ps -ef|grep agent，发现还有个 tat_agent，于是看了下 /etc/rc.d/rc.local，重启系统后发现还是有，软件咋重启呢，于是 systemctl 看了下，原来使用了系统服务，systemctl stop tat_agent 干掉它，完美。

其实腾讯云这边还是能监控到系统的使用情况的，因为这个系统应该算是一个虚拟机，是从一个物理机来划分的主机。卸载了安装的监控软件至少腾出了一点内存。没安软件之前内存剩余 788m，现在还剩 500m 左右。其实还算可以了。之前安装 docker 内存只剩 300m 左右。

对于监控，我自己写了一个 shell 脚本通过 top 命令输出到文本文件中。然后用 http 访问。刚开始 http 服务使用了 php 去拿文本，后来觉得没必要过php，换了一种思路，使用 html+iframe标签就可以(之前用的ajax，ajax不稳才用iframe)。
其实还可以通过 shell+php(我对php熟一些，如果有能力完全可以仅使用shell) 进行监控分析系统CPU，内存等，如果异常就给自己发邮件或者短信提醒，我这里就没必要了，我这里使用 shell 对于腾讯云的监控组件相比，消耗要小很多。原本 http 服务可以借助 Nginx 的 auth 做权限验证的仅持有口令才能访问，没这没弄，原本账号密码啥的就太多了，我记住不，再加上也没必要，后续再说吧。
